# Meta 更新指南

> 写给维护者：如何持续更新 `references/meta.md`，保持环境数据准确。

---

## 何时更新

| 触发条件 | 频率 | 更新范围 |
|---------|------|---------|
| 新赛季上线 | 每赛季 1 次 | 全量重写 |
| 版本平衡性补丁（削弱/增强） | 每次补丁 | 受影响的精灵和体系 |
| 新精灵/新技能加入 | 每次加入 | 个体超模榜、相关体系 |
| 新体系被开发出来 | 发现时 | 新增章节 |
| 环境明显变化（某体系泛滥/消失） | 观察确认后 | 调整评级和描述 |
| 社区对某技能的机制有新发现 | 确认后 | 更新机制描述 |

---

## 更新流程

### 第一步：收集信息源

用 bili-cli 搜索 B站 PVP 内容。**不能只看一个UP主**，至少跨 3 个来源交叉验证。

**搜索关键词模板**：

```bash
# 基础环境
bili search "洛克王国世界 S{N} PVP 环境" --type video --max 5
bili search "洛克王国世界 赛季{N} 体系 排行" --type video --max 5

# 具体体系
bili search "洛克王国世界 火队 S{N}" --type video --max 5
bili search "洛克王国世界 平衡队 阵容 S{N}" --type video --max 5

# 新精灵/热门精灵
bili search "洛克王国世界 {精灵名} 攻略" --type video --max 5
bili search "洛克王国世界 {精灵名} 配招" --type video --max 5

# 针对/反制
bili search "洛克王国世界 怎么打{精灵名}" --type video --max 5
bili search "洛克王国世界 {体系名} 克制" --type video --max 5

# 版本更新
bili search "洛克王国世界 削弱 增强 更新" --type video --max 5
bili search "洛克王国世界 平衡性调整" --type video --max 5
```

**信息源质量判断**：

| 指标 | 可信度 | 说明 |
|------|--------|------|
| 播放量 > 5万 | ★★★★ | 至少说明有观众基础 |
| UP主是 PVP 常驻创作者 | ★★★★★ | 如沐木、长夜未复、树池来了、羊阳君 |
| 实战录像（非纯讲解） | ★★★★ | 有对局记录可验证 |
| 评论区有补充和纠正 | ★★★ | 评论区经常有更好的细节 |
| 单视频单UP主独立观点 | ★★ | 必须交叉验证 |

### 第二步：提取内容

对每个视频用 `--subtitle` 获取字幕：

```bash
bili video BVxxxxxx --subtitle
```

字幕提取的关键信息：
- 体系评级和排行
- 精灵配置（完整6只+技能）
- 操作流程（首发→中期→收割）
- 克制关系和弱点
- 新精灵/新机制的描述

如果字幕不可用，用 `--ai` 获取AI摘要做补充。如果还不够，用 `--comments` 看评论区。

**注意**：字幕识别有大量错字。常见错误类型：
- 同音字：巨鼓象→巨骨象，锤头鹳→锤头冠，岚鸟→蓝鸟，棋绮后→琪琪后
- 音近字：燃薪虫→燃心虫，火焰猿→火焰源
- 形近字：食尘短绒→吃尘短绒，巨噬针鼹→巨食针鼹

### 第三步：交叉验证

同一结论必须出现在**至少 2 个独立来源**才能写入 meta.md。

验证矩阵：

```
结论：XXX体系是当前T0
├── 来源A（UP主甲，播放10万）：确认 ✗
├── 来源B（UP主乙，播放5万）：确认 ✓
├── 来源C（UP主丙，播放2万）：反对 ✗
└── 判定：2/3 确认 → 可以写入，标注为"多数共识"
```

如果只有 1 个来源 → 标记为「待验证」或暂不写入。
如果来源之间矛盾 → 在 meta.md 中同时记录两种观点。

### 第四步：wiki 验证精灵名和技能

所有精灵名和技能名必须通过 wiki API 验证正确写法：

```bash
# 验证精灵
python3 scripts/query_pet.py --name "精灵名" --skills --evo

# 验证技能
python3 scripts/query_skill.py --name "技能名"
```

如果 wiki API 返回 404 → 名字可能不对，用 B站搜索找正确写法。

如果 wiki 根本没有这个精灵的页面（如卡瓦重、棋绮后、霹雳迪迪、岚鸟）→ 用 B站社区公认名，标注「wiki未收录」。

**已知的 wiki 覆盖缺口**（这些精灵 wiki 上没有独立页面）：
- 卡瓦重（冰+草，进化链：丢丢→卡瓦虫→卡瓦重）
- 棋绮后（地武队终端收割，红桃皇后）
- 霹雳迪迪（电系传说，炫光迪迪）
- 岚鸟（欧美蓝鸟队核心C位）

### 第五步：写更新内容

更新 `references/meta.md` 时遵守：

- **体系评级**：用 T0/T1/T2/T3，不用「夯」「人上人」「NPC」等网络用语
- **每个体系至少包含**：精灵配置（6只+技能）、操作流程、弱点与反制
- **不清楚的不写**：没验证过的配招/伤害数字不要编
- **标注信息来源**：在对应段落后标注 (UP主名, BV号, 播放量)
- **保留旧版本记录**：在文末「版本变动记录」中累积

**个体超模精灵榜单**：跨UP主综合排序，至少2人认可才能上榜。

**速度线速查表**：来自新月sama、沉默的愚矢等专业机制UP主，必须注明数据来源。

### 第六步：自检清单

更新完成后逐项确认：

- [ ] 所有精灵名已验证（wiki API 或 B站搜索确认）
- [ ] 至少 3 个独立来源交叉验证
- [ ] 体系评级有 UP主 共识支撑
- [ ] 新写的内容没有和旧版本矛盾（如有矛盾，标注为「环境变化」）
- [ ] 「版本变动记录」已追加新条目
- [ ] 文件可读（没有残留的 Markdown 格式错误）

## 辅助工具

### 批量名字校验脚本

```bash
# 校验一堆精灵名是否 wiki 可查
cd ~/projects/roco-pvp-advisor
for name in 名字1 名字2 名字3; do
  result=$(python3 scripts/query_pet.py --name "$name" 2>&1 | head -3)
  if echo "$result" | grep -q "error"; then
    echo "❌ $name"
  else
    echo "✅ $name"
  fi
done
```

### 快速查某个体系的 B站讨论热度

```bash
# 看最近有没有新视频
bili search "洛克王国世界 {体系名}" --type video --max 5 --yaml | grep -E "title:|play:"
```

## 文件结构约定

```
references/
├── meta.md          ← 当前文件，环境数据
├── mechanics.md     ← 游戏机制（稳定，少改）
├── strategies.md    ← 通用策略原则（稳定，少改）
└── my-team.template.md ← 队伍模板（不改）
```

- **meta.md** 是唯一高频更新的文件
- mechanics.md 和 strategies.md 只在新机制/新策略出现时更新
- 不要在 meta.md 里重复写已在 mechanics.md 中解释过的机制（如「什么是印记」），直接引用即可
